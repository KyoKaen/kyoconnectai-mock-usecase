Business Plan Version 0.0

KyoConnectAI Startup:

Executive Summary

Section: 2. Solution

1

What do we already know?

  • We offer a customizable, knowledge-based AI chatbot that address FQAs in real time.
  • Built using OpenAI, DeepSeek, Google GenAI, Mistral APIs and deployed on Google Cloud. Technically, we validated the viability to provide high relevant answers.
  • Designed for 24/7 multilingual responses to FAQ-style customer queries.
  • One client obtained (camping site) with working MVP; demoed to others including EMLyon.
  • Envisions mid-term AI assistant to streamline more manual process, such as provide guide, long-term use as full AI agent (e.g., reservations, integration with existing CRM, check-in/out, personalized offers).
2

What do we want to know?

  • Which use cases (beyond FAQ) customers need most? How much this AI chatbot would impact customer satisfaction, attract new customers, and contribute to retention?
  • What’s the real-world cost per customer served? We need specific customers-based data. Seasonally, how many customers requested how many queries.
  • Will we need enterprise features (CRM sync, analytics dashboard)? We plan to develop dashboard for end customer preference analysis.
  • Will data preparation for customer is a big challenge? How to get the clear and expected data for chatbot? We provide 0-AI-requirement for deployment. Will customer prefer one-click to create and deploy by themselves? We plan to automate the process for creation and deployment.
3

What don’t we know?

  • Will SMBs adopt AI assistants/agents for long-term digital assets or stop at FAQ chatbots?
  • How to scale across different industries without fragmenting the codebase? For the same camping industry, technically should each chatbot be independently maintained code version or should we standardize the codebase? Low priority but need to take into consideration when architecture the project.
